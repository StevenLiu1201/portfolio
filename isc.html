My role involves :

  • Build and manage accessible training websites that meet WCAG 2.1 AA standards

  • Create engaging quizzes, interactive activities, and instructional videos to enhance learning

  • Configure course settings and troubleshoot technical issues reported by trainers and learners

  • Develop a scalable Dynamic Job Aid Framework using JavaScript with reusable components and URL-based routing

  • Perform data analysis and integrate APIs to generate customized course progress reports for internal trainers

Developed Training Courses

Several Interesting Projects

Reference guide

The Reference Guide is a searchable web-based tool designed to help students review course content more efficiently. It allows users to quickly find key information by entering keywords or phrases, making it easier to revisit specific topics or concepts. Organized for clarity and ease of use, the guide supports independent learning and reinforces key training material.

Custom Course Report

To overcome the limitations of Moodle’s built-in reporting tools, I developed a custom web-based reporting page using JavaScript. After IT restrictions prevented the use of third-party plugins, I created a solution that analyzes student log data and presents it in a user-friendly interface. The report includes sortable columns (e.g., enrollment time, exam status, exam completion time) and a search function to quickly locate individual students by name, helping trainers monitor progress more effectively.

A bunch of Job Aid Websites

I developed a standardized job aid framework to support multiple training topics, allowing all job aids to share a consistent layout and structure. By using URL parameters, each job aid page loads dynamic content while reusing the same core template. This scalable system improves maintainability and user experience. Published job aids include: Glossary Acronyms, Occupation-Based Guarantors, Acceptable and Unacceptable ID, and Name Variations on Supporting Documents, among others.
